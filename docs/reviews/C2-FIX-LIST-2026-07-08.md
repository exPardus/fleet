## FIX-1 (HIGH — F-A1/F-1) — `_resume_one_limited` under-lock re-validation
`bin/fleet.py::_resume_one_limited` unconditionally pre-claims `status="working"/turn_pid=None` inside its lock without re-checking the record is still `limited`. Every other launch path (`cmd_send` idle-resume, `cmd_respawn`) re-validates status under the claiming lock. Two racing `resume-limited` sweeps (or a sweep racing `respawn --force`/`send`) both snapshot `limited`; the second clobbers the first's live pre-claim → a second `claude --resume` on the same sid, first turn_pid orphaned.
**Fix:** inside `_resume_one_limited`'s own `fleet_lock`, re-read the record and **abort/skip if `rec.get("status") != "limited"`** (mirror `cmd_send`'s idle re-check) BEFORE pre-claiming. Also guard the `data["workers"][name]` subscript — a vanished worker must raise a clean `FleetCliError`/skip, not a raw `KeyError`. Add a test: two logical resume calls on one worker, second sees non-`limited` → skips, no double-launch (assert exactly one launch).

## FIX-2 (MEDIUM — F-A2) — tighten `_LIMIT_STDERR_RE` so a genuine crash is never mis-parked
`_LIMIT_STDERR_RE` (`usage limit|rate limit|plan limit|resets?\s+at|try again (?:later|in)|quota`) matches ordinary crash stderr: disk-quota `EDQUOT`/"Disk quota exceeded", upstream MCP `429 rate limit`, `try again later` assertions. These errored (no-result) turns get parked `limited` (null horizon → NOT auto-resumed, needs `--force-now`) AND skip `_append_abnormal_turn_note` — a real infra failure masked as a plan wall with no crash landmark.
**Fix (anchored to the classifier's role — recognize a PLAN usage-limit wall, never a crash):** tighten toward Claude's actual plan-limit phrasing. Require co-occurrence of a plan/usage-limit noun with the limit context (e.g. `(usage|weekly|session|plan|5-?hour)\s+limit`, or an explicit "limit … resets at <time>" shape), and DROP the bare `quota` / bare `rate limit` / bare `try again later` alternatives that collide with infra errors. Keep it conservative: when unsure, prefer the crash-dead path (surfaced + journal note) over a silent null-horizon park. Update the tests: the three repro crashes (disk-quota, MCP 429, retry-drained) must now classify crash-dead (with the abnormal-turn note), while a real plan-wall shape still parks `limited`. (The exact signal stays DEFERRED-TO-KERNEL-PROBE — this only removes false positives.)

Update the boundary test.
