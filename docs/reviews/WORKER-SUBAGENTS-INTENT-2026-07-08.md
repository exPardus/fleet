# Worker subagents — binding design-intent input (2026-07-08)

**Status:** binding input for Campaign 1 (spec link `spec-amend-5-subagents`) + any C2 follow-through. Authored by the fleet manager at Altai's request (2026-07-08). **No review-doc finding** — this document is the single binding input, quoted verbatim into the owning task file per PLAN §0.2.4. Plan ID: **UL2** (does not collide with review-doc M-numbers, SPEC F1–F31, or UL1).

## 1. Requirement (Altai, verbatim intent)
> "fleet workers should also be able to use their own subagents, this is another thing that should be added into the spec"

A fleet worker is a headless `claude -p` session. Claude Code sessions can spawn **subagents** (the Task/Agent tool — ephemeral child agents that run a scoped task and return a result). Altai wants workers to be able to use this like any Claude Code session, and wants the behavior specified so it is deliberate, not incidental.

## 2. What must be decided (the spec answers each; silence = spec not done)
- **UL2-OQ1 (baseline):** Is subagent use already available to a headless `claude -p` worker by default, or does anything in the fleet launch path / `worker-settings.json` disable it? PROBE the installed CLI (`claude --help`, agent/subagent flags; the `--agents`/Task-tool surface) and record evidence (§0.2.9). If it is on by default, the spec DOCUMENTS + sanctions it; if a flag is needed, name it. Do NOT invent a flag.
- **UL2-OQ2 (doctor interaction — the real hazard):** `fleet doctor`'s `_doctor_check_claude_agents` (bin/fleet.py @3101) runs `claude agents --json` and flags **fleet-unknown claude-agent sessions**. Determine and state whether a worker's ephemeral Task-tool subagents register as `claude agents` sessions (which would false-positive the doctor check) or are in-process children invisible to it. Manager's initial read: ephemeral Task subagents are children of the worker's turn, NOT registered persistent agent sessions, so they do NOT trip the check — but the spec must CONFIRM this against the CLI and state it explicitly, and if they DO appear, the doctor check must exempt a worker's own subagents (correlate by parent worker).
- **UL2-OQ3 (one-live-claude invariant, #7):** A subagent is a child process spawned by the worker's claude within its turn — NOT a competing `--resume` turn on the worker's `session_id`. State that this preserves invariant 7 (one live claude per SESSION): the subagent runs under its own ephemeral context, the worker's session is still singular. Confirm subagents do not write the worker's transcript concurrently in a way that breaks resume.
- **UL2-OQ4 (permission-mode inheritance — SECURITY):** Subagents inherit the worker's permission posture. A `bypass` worker's subagents run with the same skip-permissions power. State this plainly as a security note (it widens the blast radius of a bypass worker); reference it from the §0.4 permission doctrine. Decide whether any fleet-level guidance restricts subagent use under bypass (recommend: document, do not mechanically restrict in v1 — the worker is already trusted at bypass).
- **UL2-OQ5 (cost accounting):** Subagent token usage rolls into the worker's turn result cost. Confirm the existing cost model (§4 `cost_usd` = baseline + Σ result costs) already captures subagent spend (subagent tokens surface in the parent turn's result event), so `--max-budget-usd` / the C2 token ceiling still bound it. State it; no new schema field expected.
- **UL2-OQ6 (mid-turn steering):** fleet's PostToolUse/Stop mailbox hooks fire on the WORKER's (main agent's) tool boundaries. State that mid-turn `send` steering still lands at the main-agent boundaries (subagent-internal tool calls are nested and not individually mailbox-checked) — so steering semantics are unchanged; a worker deep in a subagent picks up mail when the subagent returns and the main agent hits its next boundary.
- **UL2-OQ7 (agent availability / foreign agents):** Like foreign hooks (§6), a worker inherits the target repo's `.claude/agents/*` and global plugin agents. State this parallel: subagent *types* available to a worker are the union of built-ins + the target repo's + global — the manager does not curate them in v1 (mirrors the foreign-hooks v1 policy).
- **UL2-OQ8 (worker preamble / doctrine):** Decide whether the §8 worker rules preamble gains a line telling workers subagents are available (and any journal-discipline note — e.g. a subagent that starts a long child process must still be recorded per the "no servers outliving the turn" rule). Recommend a short preamble/doctrine line, not a mechanism.

## 3. Concrete starting proposal (the spec refines; does not restart)
Subagent use by workers is **sanctioned and documented, not newly mechanized** — the capability is a native Claude Code feature; the fleet's job is to (a) confirm it is not disabled by the launch path/settings, (b) reconcile it with the doctor `claude agents` check so legitimate worker subagents never false-positive, (c) state the invariant-7 / cost / steering / permission-inheritance consequences, and (d) add a doctrine/preamble line. Expected code surface in v1: **little to none** — most likely a doctor-check refinement (exempt/correlate worker subagents) IF the probe shows they register, plus documentation. If the probe shows a launch flag is required to enable subagents, that flag goes into the launch path (a small C2 follow-through, noted).

## 4. Invariants touched
one-live-claude-per-session (7 — subagents are children, not competing session resumes), single-writer registry (6 — subagents never write fleet.json), exit-0 hooks (2 — unchanged), daemonless launch (1 — no resident process). The spec's `## Invariants touched` names these.
